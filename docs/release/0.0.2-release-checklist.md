# MolVis 0.0.2 Release Checklist

Manual release runbook for the `0.0.2` multi-package release.

Release scope:

- `@molvis/core`
- `page`
- `molvis` Python widget
- `MolVis` VS Code extension

## 1. Metadata Freeze

Before building, confirm:

- package versions are aligned to `0.0.2`
- visible badges and changelogs reference `0.0.2`
- license metadata is consistent across release artifacts

## 2. Build Gate

Run the monorepo build:

```bash
npm run build:all
```

This must complete successfully for:

- `core`
- `python`
- `page`
- `vsc-ext`

## 3. Test Gate

Run the full automated gate:

```bash
npm test
```

This covers:

- `core` unit tests
- `page` smoke tests
- `python` pytest smoke/regression tests
- `vsc-ext` unit tests

If needed, run targeted checks individually:

```bash
npm run test -w core
npm run test -w page
npm run test -w python
npm run test -w vsc-ext
```

## 4. Core Packaging Check

```bash
npm run release:check -w core
```

Confirm the tarball only contains release artifacts such as:

- `dist/**`
- `README.md`
- `LICENSE`
- `package.json`

## 5. Python Widget Packaging Check

The Python package requires the frontend bundle at `src/molvis/dist/index.js`.

From `python/`:

```bash
npm run build
python3 -m build --no-isolation
```

Verify both wheel and sdist are created successfully.

## 6. VS Code Extension Packaging Check

From `vsc-ext/`:

```bash
npm run package
```

Optionally produce marketplace artifacts:

```bash
vsce package
ovsx package
```

## 7. Smoke Tests

Perform quick manual checks:

- `core`: mount a minimal viewer and load a sample structure
- `page`: open the app and verify canvas, sidebars, and timeline render
- `python`: display a `Molvis` widget in a notebook and draw a frame
- `vsc-ext`: open `.pdb`, `.xyz`, and `.zarr` flows in VS Code

## 8. Release Notes

Prepare release notes that include:

- version: `0.0.2`
- package list and distribution channels
- notable user-facing changes
- known limitations
- upgrade notes, if any

## 9. Publish

Publish only after all previous steps are green.

Suggested order:

1. `@molvis/core`
2. Python widget
3. VS Code extension
4. page deployment artifact

## 10. Post-Release Verification

After publishing, verify:

- registry/marketplace version is `0.0.2`
- install from a clean environment succeeds
- quick-start examples still match the published artifacts

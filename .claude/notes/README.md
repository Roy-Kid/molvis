# Project Notes

Passive internal context for MolVis. These files survive feature completion and
accumulate over time — unlike `.claude/specs/` which are ephemeral runtime
artifacts deleted when done.

- **`notes.md`** — evolving decisions, conventions, and lessons learned.
  Written by `/mol:note`.
- **`architecture.md`** — project blueprint: modules, public surface, layer
  roles. Populated by `/mol:map`; consumed by the `librarian` agent during
  `/mol:spec`.
- **`product/`** — product strategy, vision, differentiators, iteration
  records.
- **`open-questions.md`** — things uncertain during bootstrap or design;
  user fills over time.
